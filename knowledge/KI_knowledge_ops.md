# KI: Knowledge Operations

## Обзор
Этот KI описывает внутренние инструменты и автоматизации, используемые для поддержания актуальности базы знаний проекта. Все скрипты расположены в директории `.know/scripts/`.

## Основные инструменты

| Скрипт | Описание |
|---|---|
| `audit_coverage.py` | Генерация матрицы покрытия документацией. Проверяет связь файлов с KI. |
| `sync_agents_md.py` | Обновление `AGENTS.md` на основе текущих KI и их описаний. |
| `generate_dir_index.py` | Построение индекса директорий `DIR_INDEX.md`. |
| `ki_dependency_analyzer.py` | Анализ импортов в коде и обновление ссылок между KI. |
| `knowledge_mcp.py` | MCP-сервер, предоставляющий инструменты управления знаниями для ИИ. |
| `find_unmapped_files.py` | Поиск файлов, не охваченных документацией. |
| `init_ki_system.py` | Инициализация инфраструктуры знаний в новом проекте (venv, gitignore, AGENTS.md, hardlinks). |

## Детали реализации

### Knowledge MCP (knowledge_mcp.py)
Служит мостом между внешними инструментами (например, IDE или AI-агентами) и внутренними скриптами. Он предоставляет стандартизированный набор функций (tools), позволяя агентам безопасно изменять документацию, не выходя за рамки песочницы `.know`.

### Контроль покрытия (find_unmapped_files.py)
Помогает поддерживать 100% покрытие кода. Скрипт сканирует файловую систему и проверяет наличие каждого файла в `depends_on` разделов `knowledge_items` в `doc_config.json`.
- **Игнорируемые пути**: `.git`, `__pycache__`, `node_modules`, `.venv`, `logs`, `tmp`.

### Инициализация системы (init_ki_system.py)
Скрипт автоматизирует развертывание системы знаний при клонировании репозитория в новое место или при создании нового проекта на базе шаблона.
- **Детекция Venv**: Автоматически находит путь к интерпретатору Python в `.venv`, `venv` или `env`.
- **Изоляция в Git**: Добавляет в `.gitignore` правила, которые игнорируют системные файлы `.know` (например, `ki_config.json`), но оставляют саму базу знаний (Markdown файлы) под контролем версий.
- **Инъекция инструкций**: Проверяет `AGENTS.md` и добавляет недостающие разделы: `Forced Efficiency`, `Project Navigation` и заголовок `Knowledge Items`.
- **Hard Links для Воркфлоу**: Создает жесткие ссылки из `.know/workflows/` в `.agent/workflows/` (или другую директорию IDE). Это позволяет IDE «видеть» воркфлоу, сохраняя их централизованное управление в `.know`.
- **MCP Config**: Генерирует JSON-конфиг для подключения `KnowledgeManager` к IDE в качестве MCP-сервера.

## Нюансы и ограничения
- **Кодировка**: На Windows рекомендуется использовать кодировку UTF-8 для избежания `UnicodeEncodeError`.
- **Зависимости**: Для работы скриптов требуется настроенное окружение `.venv` с установленными зависимостями из корня проекта.

## Полезные советы
- **Workflow**: При добавлении нового функционала используйте воркфлоу `/expand-knowledge`, который автоматически вызывает эти инструменты.
- **Аудит**: Если аудит показывает красные зоны, используйте `find_unmapped_files.py`, чтобы найти конкретные пропущенные файлы.
